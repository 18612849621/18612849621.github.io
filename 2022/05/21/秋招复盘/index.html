  • 单核CPU如何实现并发的呢？

    CPU的工作模式就是 “取址-译码-执行”

    程序——就是一系列指令按某种顺序在cpu上取数据进行运算

    一个程序运行的指令存储在指令寄存器中，当前运算用了哪些数据都存储在其他几个寄存器中。然后每过一段时间，对于所运行的程序进行切换（这个切换包括各种寄存器状态的存储，下一个运行程序所需寄存器的状态更新，等待再来的时候将寄存器状态拷贝回来即可），从而实现的单核CPU并发

    简单来讲就是分时 + 保存状态

    1. 调度算法

      先来先服务、短任务优先、时间片轮转

    总结一下，依靠中断来实现cpu控制权

    使用虚拟内存实现多进程的内存分配。操作系统中的内存管理策略将决定映射哪块真实的物理内存，供应用使用。这种方式也是降低了每个进程管理自己内存的复杂度，只关心如何使用自己线性排列的虚拟地址

  • 什么是虚拟内存，虚拟内存是干嘛用的？

    定义：为了更加有效地管理内存并且少出错，现代系统提供了一种对主存的抽象概念，叫做虚拟内存，虚拟内存是硬件异常，硬件地址翻译，主存，磁盘文件和内核软件的完美交互，它为每个进程提供了一个大的，一致的和私有的地址空间。通过一个很清晰的机制，虚拟内存提供了三个重要的能力：

    1. 它将主存看成是一个存储在磁盘上的地址空间的高速缓存，在主存（内存）中只保存活动区域，并根据需要在磁盘和主存之间来回传送数据，通过这种方式，它高效的使用了主存。
    2. 它为每个进程提供了一致的地址空间，从而简化了内存管理
    3. 它保护了每个进程的地址空间不被其他进程破坏

    作用：虚拟内存技术使得不同进程在运行过程中，他所看到的是自己独自占有了当前系统的4G内存。所有进程共享同一物理内存，每个进程只把自己目前需要的虚拟内存空间映射并存储到物理内存上。

    为什么要引入虚拟内存？

    在系统中所有的进程之间是共享cpu和主存这些内存资源的。当进程数量变多时，所需要的内存资源就会相应的增加。可能会导致部分程序没有主存空间可用，此外，由于资源是共享的，那么就有可能导致某个进程不小心写了另一个进程所使用的内存，进而导致程序运行不符合正常逻辑

    虚拟内存提供了三个重要的能力：缓存（击中，或者缺页），内存管理，内存保护

    1. 虚拟内存作为缓存的工具

      虚拟内存被组织为一个由存放在磁盘上N个连续的字节大小的单元组成的数组

      虚拟内存利用DRAM缓存来自通常更大的虚拟内存地址空间的页面（就是磁盘的一大块内容）

    2. 虚拟内存作为内存管理的工具

      操作系统为每个进程提供了一个独立的页表，也就是独立的虚拟地址空间。多个虚拟页面可以映射到一个物理页面上。

    3. 虚拟内存作为内存保护的工具，不应该允许一个用户进程修改他的只读段，也不允许它修改任何内核代码和数据结构，不允许读写其他进程的私有内存，不允许修改任何与其他进程共享的虚拟页面。每次CPU生成一个地址时，MMU（虚拟内存）会读一个PTE（页表）通过PTE上添加一些额外的许可位来控制一个对虚拟页面内容的访问十分简单。

    4. 虚拟内存很适合多道程序设计系统中使用，许多程序的片段保存在内存中。当一个程序等待它的一部分读入内存时，可以把CPU交给另一个进程使用。在内存中可以保留多个进程，系统并发度提高。

  • 操作系统中进程和线程的区别？

    1. 进程是资源分配的最小单元，线程是cpu调度的最小单元。
    2. 线程依赖于进程而存在，一个线程只能属于一个进程，一个进程可以有一个或多个线程
    3. 进程在执行时拥有独立的内存单元，而多个线程共享进程的内存。
    4. 进程编程调试简单可靠，但是创建销毁开销大；线程开销小，切换速度快，但是编程复杂。
    5. 进程间不会相互影响；而一个线程挂掉可能导致整个进程挂掉。

  • 线程的出现是为了解决什么问题？

    1. 在多核CPU中，利用多线程才能实现真正的并行【光进程只能是并发】
    2. 在一个应用进程中，会存在多个同时执行的任务，如果其中一个任务被阻塞，那么将会引起不依赖该任务的其他任务也被阻塞。通过对不同任务创建不同的线程处理，可以提升程序处理的实时性。
    3. 线程被认为是轻量级的进程，所以线程的创建、销毁比进程更快。

  • 常见的进程调度算法？

    1. 先来先服务：先来先服务调度算法是最简单的，每当一个进程就绪后，就加入就绪队列。当前正运行的进程停止执行，选择在就绪队列中存在时间最长的进程运行。该算法可以用于作业调度，也可以用于进程调度。比较适合长作业，不太适合短作业。
    2. 时间片轮转法：轮转法属于是抢占策略，以一个周期性间隔产生时钟中断，当中断发生后，当前正在运行的进程被置于就绪队列中，然后根据先来先服务策略区选择下一个就绪作业的运行。这个技术也称为时间片，因为每个进程在被强占前都给定一片的时间。
    3. 最短进程优先：最短进程优先是一个非抢占策略，原则是下一次选择预计处理时间最短的进程，因此短进程将会越过长作业，跳至对头。该算法即可用于作业调度，也可用于进程调度。但是不利于长作业，不能保证紧迫性进程被及时处理，作业的长短只是被估算出来的。【短进程一直进来会导致饥饿】, 不公平，而且作业时间是用户定义的，所以无法做到真正的短作业优先。

  • 先来先服务是公平的么？

    1. 是公平的、实现简单，不会产生饥饿。
    2. 排在长进程后的短进程需要等很长时间，带权周转时间很大，对于短作业用户体验不好。适合长作业。

  • Linux中用户态和内核态的区别？

    用户态：执行时，进程所能访问的内存空间和对象受到限制，其所处于占有的处理器是可被抢占的。

    1. 运行用户程序
    2. 不能直接访问操作系统内核数据结构和程序。
    3. 当我们在系统中执行一个程序时，大部分时间是运行在用户态下的，在其需要操作系统帮助完成某些它没有权利和能力完成工作时就会切换到内核态（比如操作硬件）

    内核态：执行时，能访问所有的内存空间和对象，其所占有的处理器是不允许被抢占的。

    1. 运行操作系统程序，操作硬件
    2. 系统调用、异常、外围设备中断时会跳转内核态

    1. 破坏互斥条件：

      将临界资源改成共享资源（可行性不高，很多时候无法破坏互斥条件）

    2. 破坏不可剥夺条件：

      申请的资源得不到满足时，立即释放拥有的所有资源

      申请的资源被其他进程占用时，用操作系统协助剥夺

    3. 破坏请求和保持条件：

      运行前分配好所有的资源，之后一直保持

      资源利用率低

    4. 破坏循环等待条件：

      给资源编号，按顺序进行使用

  • 知道有哪些解决哈希冲突的方式么？

    开放地址法

    线性探测：如果某数据的值已经存在，则在原来值的基础上往后加一个单位，直至不发生哈希冲突。

    再平方探测：按顺序决定值时，如果某数据的值已经存在，则在原来值的基础上先加1的平方个单位，若仍然存在则减1的平方个单位。随之是2的平方，3的平方等等。直至不发生哈希冲突。

    链式地址法

    再哈希法

    对于冲突的哈希值再次进行哈希处理，直至没有哈希冲突。

  • 一致性哈希了解过么？

    尽可能降低分布式环境下，通过哈希定位运算确认元素位置的集群服务受到机器数的影响，避免使用固定值取模，改变范围定位的方式。

    普通的负载均衡是除以机器数。

    一致性Hash的除数是2322^{32}232（int类型的最大值）。从0到2322^{32}232 - 1,首尾相连构成了一个环。

    我们先对服务器节点的IP进行Hash，然后除以2322^{32}232得到服务器节点在这个Hash环中的位置。

    如果有请求进来了，同样进行Hash然后除以2322^{32}232取余数。如果落在Hash换上，然后顺时针找到第一个节点，这个节点就负责该请求。

    优点：解决因为横向伸缩导致的大规模数据变动。

    缺点：一致性Hash算法在节点数量较少的时候，会出现分布不均匀的问题。解决方案就是在Hash环上增加虚拟节点。

  • CPU处理器从一个线程切换另一个线程需要做哪些操作？

    触发中断，通过系统调用将根据新要就绪的线程TLB保护现场并更新的寄存器内容，从而改变运行的上下文，CPU针对新的线程进行处理。是需要分模型的，如果是一对多的那种（这里指用户态多，一个内核态）那我只需要改变用户态的寄存器（对应的栈）就可以进行切换，效率很高【协程就是这样的思想】而如果是一对一的话，那么我们就需要进行变态，对于用户栈和内核栈的内容都进行更新，这样才可以进行线程的切换，开销就稍微大了一些。

  • 一个进程的阻塞不会导致另一个进程的阻塞，请问为什么？

    这个跟进程进行对比，进程之间是具有隔离性的，所以很安全不会互相破坏，用自己的（堆栈代码块全局静态存储区）当然也有共享内存（但是除了共享内存的话，每个进程之间是独立的）；但是同一个进程里的线程确实共享空间的，所以会导致不安全的问题

  • 一个线程崩掉了为什么不会导致其他线程崩掉了？

    注意一个线程只有stack 没有heap

    如果这个线程是破坏了别的线程的用stack，那么别的线程也会触发终止的信号；但是如果线程安全做的好一些的话，那么就不会出现这种情况。

  • 那为什么一个线程崩掉了会导致一个进程也崩溃呢？

    一般来说如果线程是因为非法访问内存引起的崩溃，那么进程肯定会崩溃！！！

    非法访问内存：

    1. 针对只读内存写入数据（如const常量区是只读的，如果我们对于const的变量进行修改的话，会造成非法中断）

    2. // 访问了非本进程虚拟空间的地址（这样就会有可能破坏别的进程的空间，存在危险性）
#include <stdio.h> 
#include <stdlib.h> 
int main() { 
	int *p = (int *)0xC0000fff; 
	*p = 10; // 针对不属于进程的内核空间写入数据，崩溃 
} 
<!--code￼4-->

    3. 数组越界（跟野指针是一个意思）

    以上错误都是访问内存时的错误，所以统一会报 Segment Fault 错误（即段错误），这些都会导致进程崩溃。

    为什么要让系统崩溃呢？因为这个进程中，各个线程的地址空间是共享的，既然是共享的，那么某个线程对地址的非法访问就会导致内存使用的不确定性（例如线程用户态访问了内核态的虚拟地址，所有进程共享内核态内存，导致了不安全）进而会影响其他线程或者进程。os为了避免这种危险的操作造成一系列后果，于是干脆让整个进程崩溃。
